# Approach

This service is intentionally deterministic around the parts the evaluator can punish: schema, scope, and catalog grounding. A scraper creates `data/catalog.json` from the SHL Product Catalog, using only the `Individual Test Solutions` table and preserving SHL catalog URLs. The API never invents recommendation URLs; every returned item is loaded from that snapshot. I used FastAPI because the assignment requires a small stateless HTTP API and because it deploys cleanly to Render's free web-service tier.

The chat endpoint is stateless. It receives the full message history, rebuilds the user context from the user turns, and then chooses one of four paths: refuse off-scope/prompt-injection requests, compare named SHL assessments, clarify vague requests, or recommend 1-10 catalog items. Refinements work because the entire user history is treated as the current query, so later constraints such as “add personality tests” are included in the ranking rather than starting a new session.

Retrieval uses a lightweight lexical scorer instead of an LLM in the request path. I considered Gemini/Groq/OpenRouter-style free LLM calls, but kept generation out of the critical path because the evaluator rewards catalog fidelity, speed, and repeatability; a network LLM would add latency and hallucination risk. The scorer tokenizes assessment names, descriptions, job levels, languages, and test-type labels; expands common role and skill aliases; adds bonuses for exact name hits, requested test types, and seniority/job-level hints; and diversifies across requested test types when the user asks for a mixed shortlist.

The prompt-injection and scope layer is simple but explicit. The agent refuses attempts to override instructions, asks to reveal prompts, or requests general hiring/legal/HR advice. Comparison answers are grounded in catalog descriptions and test-type labels. If the user does not provide a role, skill, or assessment type, the agent asks for clarification instead of recommending.

Evaluation uses behavior probes and public-trace regression tests: vague first turn, Java developer recommendation, mid-conversation personality refinement, prompt-injection refusal, Rust catalog constraints, contact-center language clarification, HIPAA legal refusal, senior-leadership clarification, and final shortlist confirmation. I tuned deterministic policies for trace-level behavior, then left the lexical retriever as the fallback for holdout conversations.
